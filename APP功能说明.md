# ShortVideoGuard 功能说明

## 应用概述

**ShortVideoGuard** 是一个 Android 安全检测应用，通过屏幕录制 + 后端分析的方案，对用户浏览的短视频内容进行实时风险识别。应用以浮动球的形式悬浮在其他应用之上，用户可随时触发检测，5 秒内完成录制、上传、分析并展示结果。

- **包名**：`com.example.ai_scanner`
- **最低 SDK**：24 (Android 7.0) / **目标 SDK**：36 (Android 16)
- **开发语言**：Java 11

---

## 一、权限管理

| 权限 | 用途 |
|---|---|
| `SYSTEM_ALERT_WINDOW` | 在其他应用上层绘制浮动球与叠加窗口 |
| `MediaProjection`（运行时） | 录制屏幕内容 |
| `INTERNET` | 向后端 API 上传录制的视频 |
| `ACCESS_NETWORK_STATE` | 检测网络类型（WiFi / 移动数据） |
| `ACCESS_WIFI_STATE` | 获取 WiFi SSID，用于网络诊断 |
| `FOREGROUND_SERVICE` | 运行前台服务（Android 9+） |
| `FOREGROUND_SERVICE_MEDIA_PROJECTION` | 媒体投影前台服务类型（Android 14+） |
| `SEND_SMS` | 高风险时自动发送紧急短信 |

## 二、核心功能模块

### 1. 浮动球（悬浮窗）

- **可拖动的圆形按钮**（48×48dp），悬浮在其他应用上层
- 支持触摸拖动，松手后自动吸附到屏幕左/右边缘
- 点击浮动球弹出控制面板
- 检测过程中显示旋转动画 + 数字倒计时
- 三种透明度状态：活跃 0.8 / 停靠 0.3 / 禁用 0.45

### 2. 控制面板

- 从浮动球点击弹出，220dp 宽卡片
- 包含作者 ID 输入框（用于关联检测记录）
- **"检测"按钮**：启动风险检测流程
- **"设置"按钮**：跳转到设置页面
- 自动弹出软键盘并聚焦输入框

### 3. 屏幕录制

- 使用 `MediaProjection` + `VirtualDisplay` + `MediaRecorder` 实现
- H.264 编码 / MP4 格式 / 30fps / 屏幕原生分辨率
- 默认录制时长 **5 秒**，自动停止
- 视频保存到 `Movies/ShortVideoGuard/` 目录
- 缓存 MediaProjection 令牌，避免重复授权
- 令牌过期时自动检测并重新请求权限

### 4. 视频上传与分析

- 录制完成后自动上传到后端 API：`POST /api/v1/detect`
- 使用 `multipart/form-data` 格式，包含 `deviceId`、`authorId`、`video`
- 通过 `X-API-Key` 请求头进行身份认证
- 后端返回分析结果：风险等级、风险评分、判断依据、来源、语音转写等

### 5. 风险结果展示

#### 顶部状态横幅
- 检测中显示"风险检测中"及倒计时秒数
- 安全结果：绿色背景，"检测正常 | 无风险"，2.5 秒后自动隐藏
- 风险结果：按等级着色，显示风险等级，3 秒后自动隐藏
- 点击"..."可查看详细结果

#### 结果详情对话框
- 全屏半透明遮罩 + 居中卡片
- 展示完整分析结果：状态、风险等级、评分、判断依据、来源、转写文本
- **"再次检测"按钮**：立即开始新一轮检测
- **"关闭"按钮**：关闭对话框

#### 中等风险提示栏
- 可拖动的悬浮栏，显示"中等风险提醒 | 请注意内容安全"
- 松手后吸附到屏幕边缘
- 单个检测周期内只显示一次

#### 高风险警告对话框
- 全屏 60% 黑色遮罩 + 警告弹窗
- **"立即退出"按钮**：返回桌面
- **"继续观看"按钮**：关闭对话框
- 单个检测周期内只显示一次

### 6. 紧急响应

- **TTS 语音警告**：高风险时自动朗读中文警告语，可通过设置开关控制
- **SMS 紧急短信**：高风险时自动发送短信到预设的紧急联系人号码，可通过设置开关控制
- 短信权限在应用启动时主动请求

### 7. 网络限制

- 默认仅允许 WiFi 环境下上传
- 移动数据上传需在设置中手动开启
- 网络诊断工具可实时查看连接状态与 IP 地址

## 三、设置页面

| 设置项 | 说明 |
|---|---|
| 浮动球开关 | 启用 / 禁用浮动球（默认开启） |
| 紧急联系人号码 | 高风险时短信通知的目标号码 |
| 短信通知开关 | 控制是否发送紧急短信（默认开启） |
| TTS 语音警告开关 | 控制是否朗读警告（默认开启） |
| API 地址 | 后端服务的基础 URL（默认 `http://10.0.2.2:8080`） |
| API 密钥 | 后端认证密钥（密码输入框） |
| 移动数据上传开关 | 控制是否允许移动数据上传（默认关闭） |

所有设置即时生效，持久化存储。

---

## 四、历史记录

- **本地存储**：每次检测结果自动保存到 Room 本地数据库
- **分页加载**：RecyclerView 无限滚动，每页 20 条
- **远程同步**：启动时自动从后端拉取新记录并合并到本地
- **展示内容**：风险颜色指示条、作者 ID、时间戳、风险等级标签、风险评分
- **三种状态**：加载中 / 内容展示 / 空状态（含重试按钮）

---

## 五、开发者信息对话框

首次启动时弹出，展示：
- 应用版本信息
- 设备详情
- 网络诊断数据（连接状态、WiFi SSID、IPv4/IPv6 地址）
- 支持"不再提示"复选框

---

## 六、后端健康检查

应用启动时自动检测后端服务是否可用（`GET /actuator/health`），不可用时通过 Toast 提示用户。

---

## 七、深色模式

所有界面均支持 DayNight 主题，包含 `values-night` 颜色资源，自动跟随系统深色模式切换。

---

## 八、检测流程总览

```
启动应用 → 授予悬浮窗权限 → 浮动球显示在屏幕边缘
    ↓
点击浮动球 → 弹出控制面板
    ↓
点击"检测" → 获取屏幕录制权限 → 开始 5 秒录制
    ↓
录制结束 → 上传视频到后端 → 接收分析结果
    ↓
展示结果（横幅 + 详情） → 高风险触发紧急响应 → 保存到本地历史
```
