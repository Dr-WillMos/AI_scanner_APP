# ShortVideoGuard 上线前待实现功能清单

> 基于当前代码库（2026-06-05）的全面评估。项目是一个 Android 短视频风险检测助手，通过 MediaProjection 录屏 + 后端 AI 分析实现内容审核。

## 二、Android 客户端

### 2.1 架构重构（优先级：高）

| 问题 | 说明 | 建议 |
|------|------|------|
| **单类过重** | `FloatingOverlayService` 1926 行，承载了覆盖层管理、录屏、上传、TTS、SMS 全部逻辑 | 拆分为 `OverlayManager`、`CaptureManager`、`UploadManager`、`AlertManager` 等独立类 |
| **无架构模式** | Activity/Service 直接操作所有状态，无 ViewModel/Repository 分层 | 引入 MVVM（ViewModel + LiveData/Flow） |
| **无依赖注入** | 所有依赖手动创建，测试困难 | 引入 Hilt 或 Koin |
| **手动线程管理** | `ExecutorService` + `Handler` 硬编码，生命周期管理脆弱 | 使用 Kotlin Coroutines + Flow，或至少用 WorkManager 管理上传任务 |
| **状态管理混乱** | 用十几个 boolean 标志管理状态机，扩展性差 | 使用 `sealed class` 定义显式状态 + StateFlow |

### 2.2 稳定性

| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 崩溃报告 | Firebase Crashlytics 或 Sentry | **无** — 崩溃对开发者完全不可见 |
| ANR 监控 | 主线程卡顿检测与上报 | 无 |
| 进程保活 | 前台服务被杀后自动重启 | 当前 `START_NOT_STICKY` — 被杀后不会重启 |
| 电池优化白名单 | 引导用户关闭电池优化，防止 Doze 模式杀进程 | 无 |
| 厂商适配 | 华为/小米/OPPO/vivo 等国产 ROM 的后台限制、权限管理差异 | 无 |
| WorkManager 替代 Executor | 上传任务使用 WorkManager 保证在网络恢复后重试、跨进程重启 | 当前用 `Executors.newSingleThreadExecutor()`，进程死亡即丢失 |
| 边界情况处理 | 录屏中接电话、切换应用、息屏、分屏、折叠屏等 | 部分处理（`MediaProjection.Callback.onStop()`），覆盖不全 |
| 内存管理 | 视频文件缓存的及时清理、内存泄漏检测 | 无 LeakCanary |

### 2.3 安全性

| 功能 | 说明 | 当前状态 |
|------|------|---------|
| API Key 安全存储 | 使用 EncryptedSharedPreferences 或 Android Keystore | **高危** — 明文存储在 SharedPreferences |
| HTTPS 强制 | 生产环境禁用 HTTP，仅允许 HTTPS | **高危** — `usesCleartextTraffic=true`，无网络安全配置 |
| 证书固定 (Certificate Pinning) | 防止中间人攻击 | 无 |
| 代码混淆 | ProGuard/R8 开启，保护核心逻辑 | **已关闭** — `isMinifyEnabled = false` |
| Root 检测 | 检测设备是否已 Root，提示风险或拒绝服务 | 无 |
| 模拟器检测 | 检测是否运行在模拟器上 | 无 |
| 录屏数据安全 | 录制的视频文件在本地存储的加密 | 无，存储在公共 MediaStore 中 |
| 网络安全性配置 | `network_security_config.xml` 定义可信 CA | 无 |
